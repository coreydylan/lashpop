The hydration errors you're seeing are caused by:

  1. HeroSection - uses useScroll() without layoutEffect: false
  2. SectionTransition - uses useScroll({target: ref}) before ref hydrates
  3. FounderLetterSection - uses useScroll({target: ref}) before ref hydrates

These errors happen in Next.js dev mode but NOT in production mode because:

  • Production does SSR optimization that hides these issues
  • Dev mode's Fast Refresh is more sensitive to hydration mismatches

The code on Vercel works because Vercel runs production builds
